# Transaction Feature (거래 내역 관리)

## 📌 개요

수입 및 지출 내역을 기록하고 관리하는 기능입니다.
`IncomeSource`(수입원)와 연결되어 통계 데이터의 기반이 됩니다.

## 🏗 구조

- **Controller**: `TransactionController`
  - 거래 내역 CRUD 및 필터링 조회 API 제공.
- **Service**: `TransactionService`
  - 거래 기록 생성/수정/삭제 및 **복잡한 통계 계산** 로직 포함.
  - 대시보드 및 포트폴리오 데이터 계산의 핵심 역할을 담당합니다.
- **Entity**: `Transaction`
  - 거래 데이터 모델 정의 (날짜, 금액, 유형: 수익/지출, 투입 시간 등).

## 🔑 주요 역할

1. **거래 기록**: 언제, 어떤 수입원에서, 얼마의 돈이 들어오거나 나갔는지 기록합니다.
2. **투입 시간 관리**: 해당 수익을 위해 투입된 시간을 기록하여 '시간당 수익' 계산에 활용합니다.
3. **통계 계산 엔진**:
   - **대시보드 요약**: 전월 대비 증감률 계산.
   - **월별 통계**: 연간/최근 N개월 추이 분석 (Zero-Filling 및 사용자 지정 기간 조회 지원).
   - **ROI/시간당 수익**: 투자 비용 및 투입 시간 대비 효율성을 정밀하게 분석하여 제공.
