# Transaction Feature (Frontend)

## 📌 개요

모든 수입/지출 내역을 기록하고, 리스트 형태로 조회하며 필터링할 수 있는 기능을 제공합니다.

## 🏗 주요 컴포넌트

- **TransactionList**: 거래 내역을 테이블(Table) 형태로 보여주며, 페이지네이션을 지원합니다.
- **TransactionFormDialog**: 거래 내역을 입력하거나 수정하는 폼(Form)입니다.
  - 날짜, 금액, 유형(수익/지출), 수입원 선택, 설명 등을 입력합니다.

## 🔑 주요 역할

1. **데이터 입력 최적화**: 사용자가 빠르고 정확하게 거래 내용을 입력할 수 있도록 UX를 최적화합니다 (예: 달력 위젯, 자동 완성 등).
2. **정보 조회**: 과거의 거래 내역을 기간별, 조건별로 검색하고 정렬하여 보여줍니다.
3. **시각적 구분**: 수익(+)과 지출(-)을 색상으로 구분하여 가독성을 높입니다.

## 📱 반응형 동작

- **리스트**: 모바일 환경에서는 여백이 축소되고, 긴 설명은 말줄임표(...)로 처리되어 가독성을 유지합니다.
- **입력 폼**: 모바일에서는 화면 너비를 꽉 채우는 Dialog 형태로 제공되어 입력 편의성을 높입니다.
